# Elan Registry v2.8.1 Release Plan
*Release Date: September 16, 2025*
*Type: Minor Release - Core Stability*

## 📋 Release Overview

**v2.8.1: Core Stability** focuses on menu system synchronization, code quality improvements, and enhanced development infrastructure. This release addresses critical menu management needs for v2.8.1 deployment and eliminates code duplication in administrative scripts.

---

## 🔄 **Key Features & Changes**

### Primary Features
- **Menu System Export/Import Tool** (Issue #297)
  - Environment-aware menu synchronization between development and production
  - JSON-based export format for version control compatibility
  - Automatic backup and rollback capabilities
  - Support for both Classic Menu and future UltraMenu systems
  - Integration with FIX script infrastructure

### Code Quality Improvements
- **Code Deduplication**: Eliminated duplicate functions in FIX scripts
- **Enhanced Error Handling**: Improved JSON parsing and response handling
- **Documentation Updates**: Updated CLAUDE.md with Chart.js migration details
- **Repository Cleanup**: Removed unused redirect files and references

### Infrastructure Enhancements
- **GitHub Actions Integration**: Automated menu sync workflow
- **Template-based Menu Detection**: Smart detection for future UltraMenu conversion
- **Transaction-based Imports**: Database safety with rollback capability
- **Environment Detection**: URL pattern-based environment identification

## 🛠️ **FIX Scripts Execution Plan**

### Required FIX Scripts (Execute in Order)

#### **Optional - Data Maintenance Scripts**
These scripts can be run during maintenance windows if data quality issues are detected:

1. **FIX/02-Cleanup-Orphaned-Profiles.php** *(Optional)*
   - **Purpose**: Remove orphaned user profiles without cars
   - **Impact**: Database cleanup only
   - **When to Run**: If orphaned profile reports show issues
   - **Runtime**: ~30 seconds

2. **FIX/03-Remove-Duplicate-History.php** *(Optional)*
   - **Purpose**: Clean duplicate car history entries
   - **Impact**: Historical data cleanup
   - **When to Run**: If duplicate history detected in reports
   - **Runtime**: ~1-2 minutes

3. **FIX/06-Cleanup-Orphaned-Car-User-Records.php** *(Optional)*
   - **Purpose**: Remove orphaned car-user relationships
   - **Impact**: Data integrity cleanup
   - **When to Run**: If car sharing reports show inconsistencies
   - **Runtime**: ~30 seconds

4. **FIX/09-Validate-And-Fix-Car-Image-Data.php** *(Optional)*
   - **Purpose**: Validate and fix car image references
   - **Impact**: Image data integrity
   - **When to Run**: If image display issues reported
   - **Runtime**: ~2-5 minutes depending on image count

#### **Production Deployment Scripts**

5. **FIX/10-Regenerate-Optimized-Thumbnails.php** *(RECOMMENDED FOR v2.8.1)*
   - **Purpose**: Regenerate thumbnails for optimal mobile/responsive performance
   - **Impact**: Generates new 768px thumbnails, removes old 600px thumbnails (Issue #176)
   - **When to Run**: **RECOMMENDED** during v2.8.1 deployment for image optimization
   - **Prerequisites**: None (uses existing uploaded images)
   - **Runtime**: ~5-15 minutes depending on image count
   - **Backup**: Preserves original images and existing thumbnails

6. **FIX/11-Menu-System-Sync.php** *(CRITICAL FOR v2.8.1)*
   - **Purpose**: Import menu configuration from development to production
   - **Impact**: Menu system synchronization
   - **When to Run**: **REQUIRED** during v2.8.1 deployment
   - **Prerequisites**: Export JSON from development environment
   - **Runtime**: ~30 seconds
   - **Backup**: Automatic backup of current menu system created

### FIX Script Dependencies

```
Recommended Execution Order:
1. FIX/10-Regenerate-Optimized-Thumbnails.php (image optimization)
2. FIX/11-Menu-System-Sync.php (menu synchronization)

Menu System Export/Import Flow:
1. Development: Export menu via scripts/menu-sync.php
2. Transfer: Move JSON file to production
3. Production: Import via FIX/11-Menu-System-Sync.php

Image Optimization Flow:
1. Script analyzes existing car images
2. Generates new 768px thumbnails for mobile/tablet performance
3. Removes old 600px thumbnails (underutilized)
4. Preserves all other thumbnail sizes (100px, 300px, 1024px, 2048px)
```

### FIX Script Execution Verification

After running each script, verify completion in UserSpice Admin:
- **Menu**: UserSpice Admin → System → Logs
- **Database**: Check `fix_script_runs` table for completion records
- **Files**: Verify backup files created in `FIX/backups/`

## 🚀 **Deployment Sequence**

### **Phase 1: Pre-Deployment** *(15 minutes)*
1. **Backup Current System**
   ```bash
   # Database backup
   mysqldump -u username -p elan_registry > backups/pre_v2.8.1_$(date +%Y%m%d_%H%M%S).sql

   # File backup
   tar -czf backups/files_pre_v2.8.1_$(date +%Y%m%d_%H%M%S).tar.gz app/ users/ usersc/
   ```

2. **Export Development Menu Configuration**
   - Navigate to development environment: `http://localhost:9999/elan_registry/scripts/menu-sync.php`
   - Click "Export Menus" button
   - Save JSON file: `menu_export_development_YYYYMMDD.json`

3. **Verify Dependencies**
   - [ ] PHP 8.1+ running
   - [ ] MySQL 8.0+ available
   - [ ] File permissions correct
   - [ ] Backup space available

### **Phase 2: Code Deployment** *(10 minutes)*
1. **Deploy Code**
   ```bash
   # Push to production
   git push prod main

   # Verify deployment
   curl -s -o /dev/null -w "%{http_code}" https://elanregistry.org/
   ```

2. **Update Dependencies**
   ```bash
   cd /path/to/production
   composer install --no-dev --optimize-autoloader
   ```

### **Phase 3: Database Updates** *(15 minutes)*
1. **Run Image Optimization Script** *(Optional but Recommended)*
   - Navigate to: `https://elanregistry.org/FIX/10-Regenerate-Optimized-Thumbnails.php`
   - Execute thumbnail regeneration process
   - Monitor progress (may take 5-15 minutes depending on image count)
   - Verify completion message

2. **Run Critical Menu Sync Script**
   - Navigate to: `https://elanregistry.org/FIX/11-Menu-System-Sync.php`
   - Upload the exported JSON file from Phase 1
   - Execute import process
   - Verify completion message

3. **Verify Database Changes**
   ```sql
   -- Verify FIX script completion
   SELECT * FROM fix_script_runs WHERE script_name IN ('10-Regenerate-Optimized-Thumbnails.php', '11-Menu-System-Sync.php');

   -- Verify menu data
   SELECT COUNT(*) FROM menus WHERE id > 20;
   ```

### **Phase 4: Post-Deployment Testing** *(20 minutes)*
1. **Execute Smoke Test Checklist** (see above)
2. **Verify Critical Functionality**
   - User login/logout
   - Car listings and details
   - Menu navigation
   - Statistics dashboard
3. **Check Error Logs**
   ```bash
   tail -n 50 /path/to/logs/error.log
   tail -n 50 users/logs/userspice.log
   ```

### **Phase 5: Production Verification** *(10 minutes)*
1. **Performance Check**
   - Test page load speeds
   - Verify image loading
   - Check database query performance
2. **User Access Verification**
   - Test user registration (if enabled)
   - Verify existing user access
   - Check admin panel access

## 🔄 **Rollback Plan**

### **Immediate Rollback** *(If critical issues detected within first hour)*

1. **Code Rollback**
   ```bash
   # Rollback to previous Git commit
   git revert HEAD
   git push prod main
   ```

2. **Database Rollback**
   ```bash
   # Restore from backup
   mysql -u username -p elan_registry < backups/pre_v2.8.1_TIMESTAMP.sql
   ```

3. **Menu System Rollback**
   - Navigate to FIX script backups directory
   - Use automatic backup created by menu sync script
   - Execute rollback SQL if needed

### **Graceful Rollback** *(For non-critical issues)*

1. **Identify Specific Issues**
   - Review error logs
   - Document specific problems
   - Assess impact on users

2. **Targeted Fixes**
   - Apply hotfixes for specific issues
   - Use FIX scripts to correct data problems
   - Update configuration if needed

3. **Re-deploy with Fixes**
   - Create hotfix branch
   - Test fixes in development
   - Deploy incremental update
