## Admin-Facing Changes

### Improvements

- **Owner Privilege Escalation Guard** ([#1232](https://github.com/unibrain1/elanregistry/issues/1232)): Removed `active` and `permissions` from `Owner::extractUserFields()` allowlist; changed the `validateAndSanitizeFields()` default case to drop unknown fields instead of passing them through. Prevents any future endpoint from accidentally writing these privilege-controlling columns via the general profile-update path.
- **Server Hardening** ([#1242](https://github.com/unibrain1/elanregistry/issues/1242)): Blocked web access to PHPUnit config, PHPStan config, and npm manifests; disabled HTTP TRACE; added Permissions-Policy header opting out of geolocation, camera, microphone, and payment APIs.
- **CSP Tightening** ([#1326](https://github.com/unibrain1/elanregistry/issues/1326)): Added `form-action 'self'` to the Content Security Policy (closing a form-hijacking gap that `default-src` doesn't cover); removed `unsafe-eval` from `script-src` after verifying no custom JS uses `eval()` or `new Function()`.
- **Backup Authorization** ([#1308](https://github.com/unibrain1/elanregistry/issues/1308)): Backup and restore operations now require Administrator role; editor accounts are explicitly rejected.
- **Admin Script CSRF Hardening** ([#1308](https://github.com/unibrain1/elanregistry/issues/1308)): Destructive admin maintenance scripts now require POST + CSRF token. A new shared `fix-script-core.php` centralises the auth gate, start-form, and "Close Window" button so all current and future scripts inherit the pattern automatically. "Return to Admin Console" buttons replaced with "Close Window" (closes the window and reloads the maintenance tab).
- **Admin Script Role Guard** ([#1338](https://github.com/unibrain1/elanregistry/issues/1338)): `admin_script_exec_requested()` in `fix-script-core.php` now enforces Administrator role in addition to POST + CSRF, so editor accounts cannot trigger destructive execute paths even if page-level permissions allow them to reach the script. Denied attempts are logged to the security audit log. The thumbnail maintenance script's batch continuation path (which bypasses the shared gate) receives its own `isAdmin()` guard. Also removes pre-auth `display_errors` from the thumbnail script and hardens its error reporting.
- **Login Audit Logging** ([#1243](https://github.com/unibrain1/elanregistry/issues/1243)): Confirmed that UserSpice rate-limiting is active and enforcing lockout (5 failures per account / 5 min; 20 per IP / 15 min). Added `loginFail` and `loginSuccess` security-category log entries via hooks, mirroring the logging the upstream framework ships with commented out.
- **Admin User Detail XSS Fix** ([#1306](https://github.com/unibrain1/elanregistry/issues/1306)): HTML-escaped every profile and car field rendered in the admin "manage user" hook. Closes a stored-XSS path where an owner's own `city`/`state`/`country` (or car chassis metadata) could execute in an admin session. Also guards the profile block against missing rows with a "No profile data" placeholder instead of PHP warnings.
- **DataTables Stored XSS Fix** ([#1304](https://github.com/unibrain1/elanregistry/issues/1304)): Applied `DataTables.render.text()` to every text column in the car list, factory data, and car history tables. Previously, raw database values were inserted via `innerHTML`, allowing a malicious `color` or `chassis` value to execute as XSS in any logged-in viewer's session. Also added a character-set allowlist to `ChassisValidator` (digits, letters, `/`, `-` only) that runs before the admin override branch, so override cannot be used to store HTML-injection payloads.
- **Image Filename Allowlist** ([#1307](https://github.com/unibrain1/elanregistry/issues/1307)): Validated uploaded image filenames against the `img_[32hex].(jpg|png|gif|webp)` format produced by `generateSecureFilename()` before any `glob()` or filesystem operation. Closes two paths: (1) glob hijack â€” a client-supplied `filenames=*` value that matched every in-flight upload in the shared temp directory; (2) traversal metadata oracle â€” filenames like `../../../etc/passwd` that triggered `is_file()`/`filesize()`/`exif_imagetype()` on arbitrary paths. `generateSecureFilename()` is now a method on `CarImageProcessor` and shares `ALLOWED_EXTENSIONS` with the validation logic so the two cannot drift apart. The fix script `01-Rename-Legacy-Image-Files.php` migrates the 11 pre-existing DB rows that contain legacy filenames now rejected by the allowlist (see Required Actions).
- **Concurrency Guard â€” Image Removal and Car Merge/Delete** ([#1311](https://github.com/unibrain1/elanregistry/issues/1311)): Closed three race-condition bugs in car data mutations. Image removal now uses compare-and-swap (`WHERE id = ? AND image = ?`) so two concurrent removals cannot silently overwrite each other. The admin merge transaction acquires a `SELECT â€¦ FOR UPDATE` row lock on the source car before any structural operations, preventing a concurrent delete from producing a corrupted audit trail. Car delete now checks rows-affected and throws `CarNotFoundException` on a zero-row result rather than logging a false success for an already-deleted car.
- **Admin Merge Migrated to Service Layer** ([#1349](https://github.com/unibrain1/elanregistry/issues/1349)): The bespoke merge transaction in `admin/index.php` has been replaced with a `Car::merge()` delegation. The admin page now uses the same `CarAdministrationService::merge()` path as all other callers, eliminating duplicated transaction management code and ensuring the `SELECT â€¦ FOR UPDATE` concurrency guard (from #1311) is active on the admin merge path automatically. The audit trail operation code changes from `DUPLICATE`/`NEWOWNER` to `MERGE`; the admin flash message remains unchanged.
- **Fail-Open Guard Closure** ([#1312](https://github.com/unibrain1/elanregistry/issues/1312)): Closed two fail-open paths that silently returned permissive answers on errors. The chassis-availability check now returns a server error (never "available") when the DB query fails. The transfer-request endpoint throws a user-visible error (never "No car found") on DB failure. The image-upload path guard now rejects explicitly when `realpath()` returns `false` for a missing or misconfigured image directory, closing a PHP type-coercion bug where `strpos($path, false)` returned `0` and silently skipped the traversal guard entirely.
- **Admin JS Extraction** ([#1327](https://github.com/unibrain1/elanregistry/issues/1327)): Moved all inline `<script>` blocks from admin templates to external cached JS files, eliminating `unsafe-inline` from the admin panel's effective script execution surface. The main admin JS file is renamed `admin-core.js` and gains reusable delegated event handlers (`[data-dismiss-target]`, `[data-switch-tab]`, `[data-confirm-form]`, `[data-hover-row]`, `[data-feedback-link]`) that replace a dozen single-purpose named functions. Per-owner data needed by the owner-profile form is now emitted via a structured `window.elanOwnerProfileData` island; all other tab scripts derive endpoint URLs from the existing `window.elanUrlRoot` global, eliminating PHP-injected URL strings. Six new tab-specific JS files (`design-system`, `tab-manage-cars`, `tab-account-cleanup`, `tab-owner-mgmt`, `tab-settings`, `load-owner-profile`) are minified through the esbuild pipeline and covered by ESLint. Prerequisite for removing `unsafe-inline` from `script-src` (issue #1328).
